الدرس الثاني عشر: دورة حياة المشاريع الاحترافية

الدليل الشامل للخطوات الذهبية لبناء ميزة متكاملة، بدءاً من التخطيط وحتى الـ Merge في GitHub.

المنهجية المعتمدة (The ECC Lifecycle)

لكي تضمن أن الكود الذي يكتبه الذكاء الاصطناعي احترافي، آمن، وقابل للتطوير، يجب عليك اتباع هذه الدورة (Lifecycle) بشكل صارم عند تطوير أي ميزة جديدة في مشروعك.

المرحلة 1: التخطيط الصريح (Explicit Planning)

لا تطلب كتابة الكود أبداً في رسالتك الأولى. حدد المشكلة واطلب خطة (Implementation Plan).

/ecc:plan "نريد إضافة ميزة 'سلة المشتريات' (Shopping Cart) في التطبيق. يجب أن تعتمد على Redis للسرعة. قم بإنشاء خطة تقنية توضح مسارات الـ API وشكل البيانات."

ستحصل على ملف Markdown يحتوي على الخطة. راجع الخطة، واطلب من Claude تعديلها إذا لزم الأمر قبل الموافقة.

المرحلة 2: العزل الآمن (Git Worktrees)

الذكاء الاصطناعي قد يقوم بحذف ملفات أو تغيير إعدادات عن طريق الخطأ. لا تعمل أبداً على الفرع الرئيسي (main).

استخدم git worktree لإنشاء بيئة معزولة تماماً لا تؤثر على مساحة عملك الحالية:

# في طرفية Bash الخاصة بك
git worktree add ../feature-shopping-cart shopping-cart-branch
cd ../feature-shopping-cart

المرحلة 3: دورة الـ TDD الصارمة

الآن، افتح Claude Code في المجلد الجديد واطلب تنفيذ الخطة خطوة بخطوة باستخدام الاختبارات أولاً:

tdd-workflow skill "نفذ الخطوة الأولى من الخطة (إنشاء API إضافة منتج للسلة). اكتب الاختبار أولاً ليتحقق من الفشل (RED)، ثم اكتب الكود لنجاحه (GREEN)، ثم قم بالتحسين (REFACTOR)."

المرحلة 4: المراجعات التلقائية (Code & Security Reviews)

بعد انتهاء كتابة الكود واجتياز جميع الاختبارات المحلية بنجاح، قم بتمرير الكود للوكلاء المتخصصين:

# 1. مراجعة الجودة
/code-review "راجع كود سلة المشتريات للتأكد من عدم وجود تعقيد عالي (High Cyclomatic Complexity)."

# 2. مراجعة الأمان
/security-scan "تأكد من عدم وجود ثغرات IDOR تسمح لمستخدم بمسح سلة مستخدم آخر."

المرحلة 5: التوثيق والتسليم (Git & GitHub PRs)

أخيراً، اجعل الذكاء الاصطناعي يكتب رسالة الالتزام (Commit) وملخص طلب السحب (PR Summary) نيابة عنك:

# كتابة الالتزام بالصيغة القياسية
claude "Review my staged changes and write a conventional commit message."

# إنشاء طلب السحب على GitHub (إذا كان لديك GitHub CLI)
claude "Read the commit history of this branch. Write a comprehensive Markdown PR summary including 'What Changed', 'Why', and 'How to test', then run 'gh pr create -F summary.md'."

الخاتمة

باتباع هذه الخطوات الخمسة، يتحول الذكاء الاصطناعي من مجرد "مكمل شفرة" (Auto-complete) إلى فريق هندسي كامل يعمل تحت إدارتك بدقة وموثوقية.

الدرس العاشر: بناء نظام تصميم (Design System) من الصفر

كيف تمنع الفوضى المرئية وتجعل واجهات تطبيقك احترافية ومتسقة باستخدام ECC.

الخطأ الشائع عند البدء

يطلب معظم المطورين من الذكاء الاصطناعي بناء صفحة الـ Home، ثم صفحة About، ثم صفحة Dashboard بشكل متتالي. النتيجة؟

  • زر في صفحة أحمر، وفي صفحة أخرى برتقالي.
  • خطوط بأحجام وتشكيلات لا تمت لبعضها بصلة.
  • استهلاك مفرط لسياق Claude بسبب تكرار كتابة الكود (Inline Styles).

الخطوة 1: استدعاء الوكيل المعماري (Architect)

قبل أن نكتب مكوناً واحداً، نطلب من وكيل التخطيط أو المعماري تصميم "رموز التصميم" (Design Tokens):

/ecc:plan "قم بتصميم نظام ألوان وخطوط (Design Tokens) لتطبيق تقني يتميز بالفخامة والوضع الداكن (Dark Mode). لا تقم بكتابة المكونات، فقط صمم لوحة الألوان (Palette) الأساسية والثانوية في ملف `globals.css` أو `tailwind.config.ts`."

كيف يساعد ECC هنا؟ الوكيل architect يفهم أن النظام يجب أن يمتلك متغيرات CSS مركزية (CSS Variables). سينشئ لك نظاماً متسقاً يعتمد على متغيرات مثل --color-primary و --font-heading.

الخطوة 2: إنشاء مهارة الأنماط (Design Skill)

الآن، كيف نجبر الذكاء الاصطناعي (Claude) على عدم اختراع ألوان جديدة من رأسه عند بناء الصفحات اللاحقة؟ سنستخدم مهارات ECC!

يأتي ECC بمهارة مدمجة تسمى frontend-patterns يمكن الاستفادة منها، ولكن يفضل دائماً إنشاء مهارة صارمة لمشروعك:

# إنشاء مهارة تصميم للمشروع
mkdir -p .claude/skills/my-design-system/
echo "---
name: my-design-system
description: Rules for styling any UI component in the app.
---
# إرشادات التصميم الصارمة (Design System)
1. **الألوان:** لا تستخدم ألواناً عشوائية مثل text-red-500. استخدم فقط text-primary أو bg-surface.
2. **الخطوط:** استخدم Inter للنصوص العادية، و Tajawal للعناوين.
3. **الحواف (Borders):** استخدم دائماً rounded-lg للبطاقات.
4. **المسافات:** اتبع نظام الـ 8px grid (p-4, m-2).
" > .claude/skills/my-design-system/SKILL.md

الخطوة 3: بناء المكونات الأساسية (Base Components)

الآن نحن مستعدون للبرمجة. نبدأ بمكونات الذرة (Atoms) باستخدام المهارة التي أنشأناها:

my-design-system skill "قم بإنشاء مكون `Button` في React يدعم ثلاث حالات (Primary, Secondary, Outline)."

بهذا تضمن أن الـ Button سيقرأ الإرشادات الصارمة، وسيخرج متوافقاً 100% مع هويتك البصرية، مما يجعل مظهر التطبيق "Wow" وفخم للغاية.

نصيحة إضافية: الرسوميات وتوليد الصور

إذا كنت تعمل باستخدام وكلاء ECC المدعومين بأدوات إنشاء الرسوميات (مثل توليد الصور التخيلية للواجهات عبر أدوات خارجية أو generate_image إن وجدت)، يمكنك التخطيط بصرياً قبل كتابة الكود لضمان أن التصميم المقترح يتناسب مع متطلبات العميل.

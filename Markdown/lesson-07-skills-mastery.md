## حالة استخدام: البحث العميق والمتقدم (Deep Research)

تخيل أنك تريد بناء تطبيق منافس في سوق التقنيات المالية (FinTech) وتحتاج لتحليل السوق وأفضل واجهات برمجة التطبيقات (APIs) المتاحة. بدلاً من البحث اليدوي، استخدم مهارة `deep-research`.

**كيف تنفذ ذلك عملياً؟**

```
deep-research skill "قم بإجراء بحث شامل حول أفضل واجهات برمجة التطبيقات الخاصة بالبنوك المفتوحة (Open Banking APIs) في الشرق الأوسط، وقارن بين أسعارها وميزاتها في جدول."
```

**ماذا سيحدث؟**

*   ستستخدم المهارة أدوات مثل `Firecrawl` أو `Exa MCP` للبحث في الويب الحي (Live Web).
*   ستقوم بتجميع المعلومات، تنظيمها في جدول Markdown، ووضع مراجع حقيقية (Citations) للمصادر.
*   يتم إنجاز عمل ساعات في دقائق وبدقة عالية.

## حالة استخدام: إعداد مهارة مخصصة لمعايير شركتك

لنفترض أن شركتك تمتلك إرشادات صارمة جداً حول كيفية كتابة ردود الـ API (يجب أن تحتوي دائماً على حالة `success` و `data` و `error`).

بدلاً من إخبار Claude بهذه القاعدة في كل مرة تبدأ فيها محادثة جديدة، قم ببرمجتها كـ "مهارة مخصصة".

### كيفية بناء المهارة عملياً:

1.  أنشئ مجلداً جديداً داخل مشروعك (أو في مسار `~/.claude/skills/`):  
    `mkdir -p .claude/skills/my-company-api/`
2.  أنشئ ملفاً باسم `SKILL.md` داخل المجلد.
3.  اكتب محتوى المهارة كما يلي:

```
---
name: my-company-api
description: Enforces company-wide strict JSON response format for all REST APIs.
---
# إرشادات الـ API الخاصة بالشركة

1. جميع الاستجابات الناجحة يجب أن تكون بصيغة:
{ "success": true, "data": { ... } }

2. جميع الاستجابات الفاشلة يجب أن تكون بصيغة:
{ "success": false, "error": { "code": 123, "message": "تفاصيل الخطأ" } }

3. لا تستخدم أبداً مصفوفات (Arrays) في الـ Root الخاص بـ JSON.
```

### كيفية استدعائها:

الآن، عندما تكتب كود API جديداً، كل ما عليك قوله للذكاء الاصطناعي هو:

```
my-company-api skill "أنشئ مسار API جديد لاسترجاع بيانات المستخدم."
```

سيقوم الذكاء الاصطناعي تلقائياً بفتح الملف `SKILL.md`، قراءة قواعد شركتك الخاصة، وتنفيذ الطلب ملتزماً تماماً بالتنسيق المطلوب.
